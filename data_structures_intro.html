Organising data efficiently so programs can store, access, and manipulate it fast.

How to use this guide

Topics follow the natural learning order — start with basics, then move to linear and non-linear structures. Each section shows a short description, a visual, and complexity at a glance.

1 — Fundamentals
📦

What is a Data Structure?

A way to organise and store data in memory so it can be used efficiently. Choosing the right structure is the first step to writing fast programs.

Memory Organisation Efficiency
⏱️

Time & Space Complexity

Big-O notation measures how fast an algorithm grows. O(1) is constant, O(n) is linear, O(n²) is quadratic. Always aim for the lowest possible complexity.

Big-O Performance
🗂️

Classification

Data structures split into Linear (Array, Stack, Queue, Linked List) where data is sequential, and Non-Linear (Tree, Graph) where data is hierarchical or networked.

Linear Non-Linear

3 — Non-Linear Structures
🌳

Tree

Hierarchical structure

A root node branches into children. A Binary Search Tree (BST) keeps left child < parent < right child, enabling fast search.

Key → Value mapping
