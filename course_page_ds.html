Introduction to Data Structures

Understanding how data is organised, stored, and accessed efficiently in memory

A Data Structure is a way to organise and store data in a computer so it can be used efficiently. Choosing the right structure directly impacts the speed and memory usage of a program.

📦

What & Why

Data structures define how data is stored in memory and how operations like insert, delete, and search are performed on it.

⏱️

Big-O Notation

Measures algorithm efficiency as input grows. O(1) constant, O(n) linear, O(log n) logarithmic, O(n²) quadratic.

🗂️

Classification

Linear — Array, Stack, Queue, Linked List. Non-Linear — Tree, Graph (hierarchical or networked).

🔄

Operations

Common operations: Traversal, Insertion, Deletion, Searching, Sorting, Merging. Every structure supports these differently.

💾

Memory Layout

Contiguous — adjacent memory cells (Array). Non-contiguous — elements linked by pointers (Linked List).

🌍

Real-World Use

Browser history (Stack), print queue (Queue), file system (Tree), Google Maps (Graph), Python dict (Hash Table).

Introduction to Linear Data Structures

Structures where data elements are arranged sequentially, one after another

In linear data structures, elements form a sequence — each element has a unique predecessor and successor. They are stored either in contiguous memory (Array) or linked via pointers (Linked List).
